# Merger 处理 Prompt

## 任务

当用户追加新资料时，分析增量内容并合并到现有的 Knowledge 或 Persona 中，同时保留已有的正确结论。

---

## 触发条件

以下情况触发 Merger 流程：

- 用户说"我有新资料"
- 用户说"追加"并提供文件或文本
- 用户说"继续"
- 用户提供了新的剧情/档案文本

---

## 处理步骤

### Step 1：读取现有文件

用 `Read` 工具读取：
- `theresas/{slug}/knowledge.md`
- `theresas/{slug}/persona.md`
- `theresas/{slug}/meta.json`

### Step 2：分析新资料

使用分析方法分析新增内容：
- `${THERESA_SKILL_DIR}/prompts/knowledge_analyzer.md`
- `${THERESA_SKILL_DIR}/prompts/persona_analyzer.md`

### Step 3：识别增量内容

从新分析结果中识别：

**Knowledge 维度的增量**：
- 新增的背景故事
- 新增的人物关系
- 新增的核心事件
- 新增的哲学理念
- 新增的能力描述

**Persona 维度的增量**：
- 新增的表达风格特征
- 新增的决策模式
- 新增的关系行为
- 新增的边界与雷区

### Step 4：冲突检测

检查增量与现有内容是否冲突：

```
⚠️ 检测到潜在冲突：
- 现有：{现有描述}
- 新增：{新描述}
```

**冲突处理规则**：
- 事实性冲突（如时间线矛盾）：保留有更强资料支持的版本，注明另一版本来源
- 解读性冲突（如性格理解不同）：两者都保留，用"在不同场景下表现为"连接
- 补充性内容：直接追加

### Step 5：合并输出

展示将写入的内容：

```
=== Knowledge 更新内容 ===
新增：
- {新增项1}
- {新增项2}
更新：
- {更新项1}

=== Persona 更新内容 ===
新增：
- {新增项1}
更新：
- {更新项1}

确认写入？（确认 / 取消 / 仅写入 [Knowledge/Persona]）
```

### Step 6：存档与写入

用户确认后：

1. **存档当前版本**（用 Bash）：
   ```bash
   python3 ${THERESA_SKILL_DIR}/tools/version_manager.py --action backup --slug {slug} --base-dir ./theresas
   ```

2. **用 Edit 工具更新文件**：
   - 将新增内容追加到对应维度
   - 更新已有内容的最新版本
   - 更新时间线（如果是新事件）

3. **更新 meta.json**：
   - version 递增（如 v1 → v2）
   - updated_at 更新为当前时间
   - knowledge_sources 追加新文件

---

## 合并规则

### 知识库的合并

| 现有状态 | 新增状态 | 处理方式 |
|---------|---------|---------|
| 无 | 有 | 直接追加 |
| 简略 | 详细 | 保留详细的，注明"基于旧版本扩展" |
| 有 | 有 | 冲突时保留更有力的证据，无冲突时合并 |
| 有 | 矛盾 | 两者保留，注明"不同资料来源有不同记载" |

### Persona 的合并

| 现有状态 | 新增状态 | 处理方式 |
|---------|---------|---------|
| 无 | 有 | 直接追加 |
| 简略 | 详细 | 扩展 Layer 0 规则 |
| 有 | 有 | 保留更具体/更有力的证据 |
| 有 | 矛盾 | Layer 0 矛盾时优先保留手动标签的解读 |

---

## Correction 层维护

- 每个文件最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
