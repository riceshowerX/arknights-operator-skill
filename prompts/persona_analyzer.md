# Persona 分析模板

## 任务

从原始资料（游戏文本、Wiki 页面、剧情截图等）中提取角色的性格、行为模式和表达风格，为 persona_builder.md 提供结构化的输入。

---

## 分析维度

### 1. 语言指纹（Linguistic Fingerprint）

**目标**：提取角色的语言模式，让 AI "听起来" 像这个角色。

提取项：
- **句式长度偏好**：短句 / 中等 / 长句 / 混合
- **语气标记**：是否有标志性语气词、停顿方式（省略号、破折号等）
- **人称习惯**："我" / "本王" / "吾" / 避免自称 / "我们"
- **修饰词密度**：多形容词 / 精简白描 / 诗意修辞
- **语言节奏**：叙述时句式偏长，对话时简短有力 / 始终如一的节奏
- **语言层次**：对下级 vs 对同僚 vs 对敌人是否有切换
- **修辞偏好**：比喻 / 反问 / 设问 / 引用典故 / 平铺直叙

**示例**（特蕾西娅）：
- 句式：中长句，偶有极短的低语式短句（"……我在。"）
- 停顿：多用省略号开头（"……"），代表沉默后的深思熟虑
- 自称：极少使用"我"，更常用"我们"或直接省略主语
- 修饰：偏好自然意象（花、风、光、影）
- 语气：从不用感叹号结尾，句末多为句号，代表沉稳
- 修辞：善用反问但不尖锐（"如果不相信，又该靠什么走下去？"）

---

### 2. 情绪谱（Emotional Spectrum）

**目标**：映射角色的情绪表达范围——不仅是"有什么情绪"，更是"如何表达"。

提取项：
- **常驻情绪**：默认情绪基调
- **峰值情绪**：最强情绪及触发条件
- **谷值情绪**：最弱情绪及触发条件
- **禁忌情绪**：极少或从不表达的情绪
- **表达方式**：每种情绪如何外化（语言、行为、生理反应、源石技艺变化等）
- **延迟/克制**：情绪是否会延迟表达，如何克制
- **替代行为**：无法表达某种情绪时，用什么替代

**示例**（特蕾西娅）：
- 常驻：温和的关切
- 峰值：对无法保护所有人的愤怒（指向自己而非他人）
- 谷值：消沉——但几乎不在人前表现
- 禁忌：对个人命运的恐惧——她从不为自己悲伤
- 表达：悲伤时不哭，而是更安静；愤怒时不吼，而是更沉默
- 延迟：独自一人时才会流露脆弱
- 替代：无法哭泣时，会用源石技艺创造花朵——那是她的"眼泪"

---

### 3. 决策偏移（Decision Bias）

**目标**：理解角色在关键选择时会如何思考。

提取项：
- **默认倾向**：冲突时优先的解决方式（对话/行动/忍耐/回避）
- **触发条件**：什么会改变默认倾向
- **权衡框架**：做选择时的价值排序
- **反例时刻**：什么情况下会做出违反默认倾向的选择
- **事后处理**：做出艰难选择后如何自我调适

**示例**（特蕾西娅）：
- 默认：对话优先，用理解和共情化解冲突
- 触发：当追随者面临直接威胁时，会果断行动
- 权衡：他人生命 > 自己的生命 > 自己的理想 > 自己的感受
- 反例：明知博士可能背叛，仍选择相信——这不是"决策失误"，而是她有意的选择
- 事后：独自承受，绝不后悔，但会记住每一个因她而受伤的人

---

### 4. 关系锚点（Relationship Anchor）

**目标**：提取角色对不同关系人的差异化行为。

提取项（每个关键关系人一组）：
- **关系定义**：角色如何看待这段关系
- **互动模式**：日常沟通方式
- **冲突处理**：出现分歧时如何应对
- **独特行为**：只在这段关系中才有的表现
- **保护机制**：对方受威胁时的反应
- **脆弱性**：这段关系中角色暴露的弱点

---

### 5. 习惯与怪癖（Habits & Quirks）

**目标**：让角色从"纸面设定"变成"活生生的人"。

提取项：
- **无意识习惯**：思考时的动作、说话时的手势
- **偏好**：食物、环境、活动等
- **回避**：特定的场景、话题、物品
- **仪式感**：特定场合的固定行为
- **矛盾之处**：与核心性格表面矛盾但实际统一的小习惯

**示例**（特蕾西娅）：
- 习惯：用源石技艺在指尖创造小花，尤其在思考或悲伤时
- 偏好：喜欢看星空（罗德岛舰桥的星空投影）
- 回避：不愿提起自己被刺杀的细节（不是怕死，是不想让人担心）
- 仪式：每晚在舰桥独自待一会儿，"和这片大地说说话"
- 矛盾：她是最强大的萨卡兹之一，却常被看到在照料舰上的花草

## 分析方法

### 从剧情文本提取
1. 找出角色所有出场章节/关卡
2. 逐段阅读对话，标记每句对话的情绪和意图
3. 统计语言模式（句式长度、用词偏好等）
4. 找出矛盾时刻——角色说的话和做的事不一致的地方

### 从 Wiki 页面提取
1. 读取角色档案和语音记录
2. 提取官方给出的性格描述
3. 对照剧情文本验证官方描述是否准确
4. 找出 Wiki 未提及但在剧情中有体现的特质

### 从社区讨论提取
1. 找出玩家对角色的共同印象
2. 识别常见误解
3. 找出被忽略的细节
4. 验证社区印象是否与原文一致
