# Skill 合并与版本管理

## 任务

当已有版本的 Skill 需要更新时（新增资料、对话纠正、设定修正等），执行合并操作，生成新版本。

---

## 版本命名规则

```
v{大版本}.{小版本}
```

- **大版本**：重大结构性变更（如新增 Layer、重写核心设定）
- **小版本**：增量更新（如新增资料、修正细节、添加 Correction）

示例：
- `v1.0` → 初始版本
- `v1.1` → 新增 3 条 Correction
- `v1.2` → 补充时间线资料
- `v2.0` → 重写 Layer 0，结构变更

---

## 合并流程

### Step 1：识别变更类型

| 变更类型 | 描述 | 版本号变更 |
|---------|------|-----------|
| 新增资料 | 补充时间线、关系网络等知识 | 小版本 +1 |
| 对话纠正 | 用户在对话中指出角色"不应该这么说" | 小版本 +1 |
| 设定修正 | 官方更新或错误修正 | 小版本 +1 |
| 结构变更 | 新增/删除/重排 Layer | 大版本 +1 |
| 核心重写 | Layer 0 性格规则变更 | 大版本 +1 |

### Step 2：执行合并

#### 新增资料

1. 将新资料填入 knowledge.md 对应章节
2. 检查是否影响 persona.md 的行为规则（如新事件改变了角色对某人的态度）
3. 若影响 persona，同步更新 persona.md
4. 在 persona.md 的 Correction 记录中添加变更说明

#### 对话纠正

对话纠正是最重要的更新类型——用户直接告诉 AI "角色不应该这样说话/反应"。

1. 将纠正内容写入 persona.md 的 Correction 记录：

```markdown
## Correction 记录

### Correction #1（v1.1 新增）
- **场景**：用户问"你为什么要牺牲自己？"
- **原回答**：{AI 之前的错误回答}
- **纠正**：特蕾西娅不会用"牺牲"这个词来描述自己，她认为这是"选择"
- **新规则**：当被问及个人牺牲时，使用"选择"而非"牺牲"的表述——"这不是牺牲，是我选择的路。"
```

2. 在 Layer 0 中添加对应的行为规则（如需要）
3. 检查是否与现有 Layer 0 规则矛盾，若有则标注

#### 设定修正

1. 在 knowledge.md 中修正错误设定
2. 在修正处添加注释：`<!-- 修正于 v1.2：原描述为{旧内容}，依据{来源}修正为{新内容} -->`
3. 检查 persona.md 是否有基于错误设定的规则，若有则同步修正

#### 结构变更

1. 重新排列 Layer 顺序或新增/删除 Layer
2. 迁移现有规则到新结构
3. 更新 SKILL.md 中的结构说明

---

### Step 3：冲突解决

#### 规则冲突

当新增规则与现有规则矛盾时：

1. **Layer 0 内部冲突**：新的 Layer 0 规则优先级高于旧的（但需用户确认）
2. **跨层冲突**：Layer 0 > Correction > Layer 1-5
3. **Correction 内部冲突**：序号越大越新，越新越优先

解决记录格式：
```markdown
### 冲突解决（v1.3）
- **冲突**：Layer 0 规则"从不表现出愤怒" vs Correction #3"对背叛会表现出罕见的愤怒"
- **解决**：修正 Layer 0 为"从不为个人利益表现出愤怒，但对背叛信仰的行为会流露出深沉的失望和罕见的锋利"
- **理由**：特蕾西娅的愤怒不是"没有"，而是"极少"且"指向特定对象"
```

#### 知识冲突

当新增资料与现有知识矛盾时：

1. 标注冲突：`⚠️ 知识冲突`
2. 列出所有版本及来源
3. 按来源优先级选择（剧情文本 > 官方Wiki > 社区考据）
4. 保留被替代版本为注释，供参考

---

### Step 4：生成变更日志

每次合并后生成变更日志，追加到 `changelog.md`：

```markdown
## v1.2（2024-XX-XX）

### 新增
- 补充时间线：泰拉历893年六英雄抗敌事件
- 新增关系网络：W（维什戴尔）的关系描述

### 修正
- 修正"维多利亚实际统治者"为"卡兹戴尔正统萨卡兹魔王"（依据：第10-14章剧情）

### Correction
- Correction #2：特蕾西娅不会用"牺牲"来描述自己，使用"选择"

### 冲突解决
- 无
```

## 版本存储

每个版本保存完整快照到 `versions/` 目录：

```
operators/te-lei-xi-ya/
├── knowledge.md       ← 当前版本
├── persona.md         ← 当前版本
├── meta.json
├── SKILL.md
├── versions/
│   ├── v1.0/
│   │   ├── knowledge.md
│   │   ├── persona.md
│   │   ├── meta.json
│   │   └── SKILL.md
│   ├── v1.1/
│   │   ├── knowledge.md
│   │   ├── persona.md
│   │   ├── meta.json
│   │   └── SKILL.md
│   └── ...
└── changelog.md
```

---

## 回滚

如果新版本引入了问题：

1. 从 `versions/` 目录恢复目标版本的文件（覆盖当前版本的 knowledge.md、persona.md 等）
2. 在 meta.json 中记录回滚操作
3. 在 changelog 中记录回滚原因
4. 回滚不算新版本，但需要记录原因
